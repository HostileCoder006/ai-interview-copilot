# AI Copilot for Technical Interviews

An AI interview practice app for technical interview preparation. Supports text and camera/voice interview modes, AI-powered evaluation, saved session history, and detailed feedback.

## Features

| Feature              | Description                                                                      |
| -------------------- | -------------------------------------------------------------------------------- |
| Interview Simulator  | Alex asks DSA, system design, OOP, and behavioral questions.                     |
| Voice Interview Mode | Records spoken answers, transcribes via Whisper, evaluates via DeepSeek.         |
| AI Evaluation        | Scores correctness, communication, confidence, optimization, and missing points. |
| Interview History    | Saves completed sessions in localStorage.                                        |
| Analytics            | Shows score trends and session summaries.                                        |

## Tech Stack

| Layer         | Technology                                                   |
| ------------- | ------------------------------------------------------------ |
| Framework     | TanStack Start                                               |
| UI            | React 19 + TypeScript                                        |
| Styling       | Tailwind CSS v4                                              |

## Voice Interview Architecture

The voice interview feature (`/voice`) works fully online — no local Python server required:

1. **Frontend** — `MediaRecorder` captures mic audio while the candidate answers
2. **Submit** — audio blob is POSTed to `/api/transcribe-and-evaluate`
3. **Backend step 1** — calls OpenRouter `openai/whisper-large-v3` to transcribe the audio (base64 JSON, not multipart upload)
4. **Backend step 2** — sends the transcript to `deepseek/deepseek-v4-flash` for evaluation
5. **Response** — returns `{ transcript, evaluation }` with scores, feedback, and a follow-up question

Live interim captions are shown via the browser Web Speech API while recording (display only — the final transcript always comes from Whisper).

## API Key Setup

Create a `.env` file in the project root (copy from `.env.example`):

```env
OPENROUTER_API_KEY=your_openrouter_api_key_here

# Optional: different keys for speech vs evaluation
OPENROUTER_TRANSCRIPTION_API_KEY=your_key_for_whisper
OPENROUTER_EVALUATION_API_KEY=your_key_for_deepseek
```

Get keys at [openrouter.ai/keys](https://openrouter.ai/keys). Whisper and DeepSeek are both billed through OpenRouter — you do not need separate OpenAI/DeepSeek accounts.

## Important Files

| File                                  | Purpose                                              |
| ------------------------------------- | ---------------------------------------------------- |
| `src/routes/voice.tsx`                | Camera interview UI                                  |
| `src/hooks/useOnlineTranscription.ts` | MediaRecorder → `/api/transcribe-and-evaluate` hook  |
| `src/server.ts`                       | API endpoints (transcribe + evaluate in one request) |
| `src/lib/api-config.ts`               | OpenRouter base URL, model names, fallback API key   |
| `src/lib/mock-evaluator.ts`           | Local fallback evaluator (used if API call fails)    |
| `src/lib/interview-storage.ts`        | Saves and loads local interview history              |

## Notes

- `src/routeTree.gen.ts` is generated by TanStack Router. Do not edit it manually.
- `dist/` and `node_modules/` are generated folders.
- Use npm for this project to keep dependency management simple.
- No Python or separate backend process — API routes run in the TanStack Start server (`src/server.ts`) via the Cloudflare Vite plugin during development.
